# archive-portal-app

本仓库用于归档和发布多个独立 HTML 页面，可以作为轻量级的页面存档与展示入口。

## 项目简介

archive-portal-app 是一个面向存档场景的静态页面集合仓库。它不绑定任何特定域名或网站，旨在将不同来源或用途的 HTML 页面统一收纳，便于后续访问与维护。适合个人或团队用于保存临时页面、离线演示、历史快照等场景。

## 目录说明

```
.
├── index.html          # 入口页面，提供归档列表或导航
├── pages/              # 存放所有归档的 HTML 页面
│   ├── example-page1.html
│   ├── example-page2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── style.css
│   └── script.js
└── README.md
```

- **index.html**：仓库的首页，可展示归档页面的索引或简要说明。
- **pages/**：所有需要存档的独立 HTML 页面均放置于此目录下，每个页面保持独立完整。
- **assets/**：存放页面共享的静态资源，如样式文件、脚本文件或图片素材。

## 页面归档说明

- 每个归档页面应为一个独立的 HTML 文件，包含完整的 `<!DOCTYPE html>` 声明及必要元数据。
- 建议在页面内部注明存档日期、来源说明或版本信息，以便追溯。
- 页面之间应避免硬编码绝对路径，推荐使用相对路径引用公共资源。
- 如需新增归档页面，直接在 `pages/` 目录下创建新文件，并更新 `index.html` 中的索引列表。

## 维护说明

- 本仓库不提供后端服务或动态生成功能，所有页面均为静态 HTML。
- 更新或删除页面时，请同步修改 `index.html` 中的导航链接。
- 建议定期检查归档页面的链接有效性及资源引用情况。
- 如仓库用于协作，请通过 Pull Request 提交变更，并附带简要变更说明。
